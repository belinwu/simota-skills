# PDF Accessibility Anti-Patterns

> PDF アクセシビリティの失敗パターン、PDF/UA 準拠ミス、WCAG 違反、自動修復の限界

## 1. PDF アクセシビリティ 7 大アンチパターン

| # | アンチパターン | 問題 | 影響 | 対策 |
|---|-------------|------|------|------|
| **PA-01** | **タグなし PDF** | 構造タグがないPDFを生成 | スクリーンリーダーが見出し・段落・リストを識別不能 | 全要素に適切なタグ付け必須 |
| **PA-02** | **読み順序の未定義** | 論理的な読み順序が設定されていない | コンテンツが不正確な順序で読み上げられる、スキップ | タブ順序と読み順序を明示的に設定 |
| **PA-03** | **代替テキストの欠如/低品質** | 画像に alt テキストなし、または「image.png」的な無意味テキスト | 視覚障害者が画像内容を理解不能 | 意味のある代替テキスト + 装飾画像はアーティファクトタグ |
| **PA-04** | **テーブル構造の未定義** | テーブルヘッダー未設定、セル関係未定義 | 複雑なテーブルのナビゲーション不能 | 行/列ヘッダーの明示的マークアップ |
| **PA-05** | **フォーム要素のラベル欠如** | フォームフィールドにプログラム的ラベルなし | 入力フィールドの目的が不明 | name/role/value を全フォーム要素に設定 |
| **PA-06** | **コントラスト不足** | テキストと背景のコントラスト比が不十分 | ロービジョンユーザーが読み取り困難 | WCAG 1.4.3: 通常テキスト 4.5:1、大文字 3:1 |
| **PA-07** | **自動修復への過度依存** | 自動ツールのみでアクセシビリティ対応 | 構造的問題（読み順序・テーブルヘッダー・alt テキスト品質）の見逃し | 自動スキャン + 手動レビューのハイブリッド |

## 2. WCAG PDF 技法（23 の要件）

```
WCAG PDF 23 技法の分類:

  構造要素（6 技法）:
    PDF1:  画像の Alt エントリによるテキスト代替
    PDF2:  ナビゲーション用ブックマーク
    PDF3:  正しいタブ・読み順序
    PDF4:  装飾画像のアーティファクトタグ
    PDF9:  見出しタグによるドキュメント構造
    PDF21: リストタグによる適切なリストマークアップ

  フォームアクセシビリティ（6 技法）:
    PDF5:  必須フォームコントロールのマーキング
    PDF10: インタラクティブコントロールのラベル
    PDF12: フォームフィールドの name/role/value
    PDF15: 適切なアクションを持つ送信ボタン
    PDF22: 入力バリデーションフィードバック
    PDF23: インタラクティブフォームコントロール（静的でない）

  コンテンツ拡張（11 技法）:
    PDF6:  テーブル要素の適切なマークアップ
    PDF7:  スキャン文書の OCR
    PDF8:  略語の定義
    PDF11: 注釈による説明的リンクテキスト
    PDF13: リンクの代替テキスト
    PDF14: ランニングヘッダー/フッター
    PDF16: デフォルト言語指定
    PDF17: 一貫したページ番号
    PDF18: ドキュメントタイトルのメタデータ
    PDF19: 多言語パッセージの言語タグ
    PDF20: Adobe ツールによるテーブル修復
```

## 3. PDF/UA（ISO 14289）準拠ガイド

```
PDF/UA 準拠の必須要件:

  タグ付け:
    → 全テキスト、画像、テーブルに正確なタグ
    → 支援技術がコンテンツを正確に解釈可能
    → タグの階層構造が論理的

  メタデータ:
    → ドキュメントタイトル
    → 言語指定
    → コンテンツ説明

  フォント:
    → 全フォントの埋め込み必須
    → デバイス間で一貫した表示

  インタラクティブ要素:
    → フォームフィールドのアクセシビリティ
    → キーボードナビゲーション対応

  コンプライアンス期限（2025-2026）:
    → 2025年6月28日: EU EAA 準拠期限（WCAG 2.1 AA）
    → 2026年4月24日: 米国 ADA Title II 準拠期限（人口5万以上の自治体）

  ❌ アンチパターン: スキャン PDF をそのまま公開
    → 画像のみで構造情報ゼロ
    → OCR → タグ付け → 検証の全工程が必要

  ❌ アンチパターン: 印刷ツールからの直接エクスポート
    → PDF は HTML の動的タグ付けを欠く
    → 後付けのタグ付けは労力大

  ✅ 推奨: ソースからアクセシブルに生成
    → Markdown/HTML → Tagged PDF の直接変換
    → 変換時にタグ構造を自動生成
```

## 4. アクセシビリティ検証チェックリスト

```
PDF アクセシビリティ検証の 4 レイヤー:

  Layer 1: 自動検証
    □ PDF/UA バリデーター（PAC 2024 等）でスキャン
    □ 全 23 WCAG PDF 技法の自動チェック
    □ コントラスト比の自動計測
    □ タグ構造の整合性チェック

  Layer 2: 手動検証
    □ 読み順序の論理性確認
    □ 代替テキストの意味的適切性
    □ テーブルヘッダーの正確性
    □ フォームラベルの明確性

  Layer 3: 支援技術テスト
    □ スクリーンリーダー（NVDA/JAWS）でのナビゲーション
    □ キーボードのみでの操作確認
    □ 拡大表示での可読性

  Layer 4: ユーザーテスト
    □ 実際の支援技術ユーザーによるレビュー
    □ フィードバックの反映
```

## 5. Morph との連携

```
Morph での活用:
  1. CONVERT フェーズで PA-01〜07 のスクリーニング
  2. accessibility-guide.md と連携した WCAG 23 技法の適用
  3. VERIFY フェーズでアクセシビリティ検証チェックリスト適用
  4. quality-assurance.md と連携した品質スコアにアクセシビリティ指標を統合

品質ゲート:
  - タグなし PDF 出力 → タグ付け必須（PA-01 防止）
  - 読み順序未設定 → 明示的設定必須（PA-02 防止）
  - 画像 alt テキストなし → 代替テキスト必須（PA-03 防止）
  - テーブルヘッダー未定義 → マークアップ必須（PA-04 防止）
  - コントラスト比未確認 → 4.5:1 検証必須（PA-06 防止）
  - 自動ツールのみ → 手動レビュー追加必須（PA-07 防止）
  - PDF/UA 準拠未確認 → PAC 等での検証必須
```
