# Steering Prompt Anti-Patterns

> NotebookLMステアリングプロンプト設計の失敗パターン、プロンプトエンジニアリングの罠

## 1. ステアリングプロンプト 7 大アンチパターン

| # | アンチパターン | 問題 | 兆候 | 対策 |
|---|-------------|------|------|------|
| **SP-01** | **Vague Steering（曖昧ステアリング）** | 「良い感じにまとめて」「わかりやすく」等の主観的指示 | 出力が一般的で焦点がぼやける、毎回異なる品質 | 具体的制約を付与: 対象者×フォーカス×トーン×長さを明示 |
| **SP-02** | **Prompt Bloat（プロンプト肥大化）** | 無関係な情報や過剰な詳細を詰め込む | LLMの推論が3,000トークン付近で劣化、「中間の情報が消失」する現象 | 150語以内、指示は8つ以下、不要な文脈を除去 |
| **SP-03** | **Multi-Task Overload（マルチタスク過負荷）** | 1つのプロンプトに複数の無関係なタスクを詰め込む | 品質が全タスクで均等に低下、焦点が分散 | 1プロンプト=1目的、複雑なタスクはステップに分割 |
| **SP-04** | **Audience Omission（オーディエンス省略）** | ターゲットの知識レベル・関心・文脈を指定しない | 出力が一般的、専門家向けが初心者向けに、またはその逆 | Audience Anchorパターン: 冒頭で対象者を明示的に定義 |
| **SP-05** | **Positive-Only Framing（肯定のみの指示）** | 「何を含めるか」だけで「何を省くか」を指示しない | 不要な概要・定義・免責事項が含まれる、冗長な出力 | Negative Spaceパターン: 明示的に除外事項を列挙（"Skip: ..."） |
| **SP-06** | **Contradictory Instructions（矛盾する指示）** | 「簡潔に」と「包括的に」のように相反する指示 | LLMが一方を優先し他方を無視、不安定な出力 | 指示間の整合性を事前チェック、優先順位を明示 |
| **SP-07** | **Customization Box Confusion（設定欄の混同）** | Audio Overviewのステアリングをチャット欄に入力 | ステアリングが反映されない、通常のQ&A応答になる | Audio/Video: 「Customize」ボックスに入力、チャットは質問用 |

## 2. プロンプト構造の罠

```
構造設計の失敗:

  ❌ No Role Assignment（ロール未指定）:
    → AIがデフォルトの汎用アシスタントモードで動作
    → 出力のトーン・深度・視点が不安定
    → 対策: ペルソナを指定（「7年生の先生として」「CTOの視点で」）

  ❌ Format Ambiguity（フォーマット曖昧）:
    → 出力形式を指定しない
    → 構造化されていないテキストブロックが返る
    → 対策: 明示的な構成指示（「3セクション構成で」「箇条書き5点で」）

  ❌ No Examples（例示なし）:
    → 複雑なタスクで期待する出力パターンを示さない
    → AIが一般的なトレーニングデータにフォールバック
    → 対策: 1-2の具体例を含める（Few-shot prompting）

  ❌ Missing Delimiter（区切り文字なし）:
    → 指示とソース内容が混在
    → AIが指示をコンテンツとして扱う混乱
    → 対策: 三重バッククォート、XMLタグ等で指示と内容を分離

  ❌ Length Neglect（長さ未指定）:
    → 出力の長さが毎回大きく変動
    → 長すぎてポイントが埋もれる、短すぎて情報不足
    → 対策: Duration Targetパターン: 「厳密に10-12分」「100語以内」
```

## 3. NotebookLM固有の罠

| # | アンチパターン | 問題 | 兆候 | 対策 |
|---|-------------|------|------|------|
| **NB-01** | **Generic Summarization（単純要約依存）** | 「まとめて」だけでNotebookLMの分析力を無駄に | ソースの表面的なリフレーズのみ、深い洞察なし | 特定の分析角度を指定: 矛盾検出、ギャップ分析、批判的評価 |
| **NB-02** | **Citation Neglect（引用要求の省略）** | ソースからの引用・根拠を要求しない | ハルシネーションや一般論が混入、検証不能な主張 | 「すべての主張をソースからの直接引用で裏付けて」を含める |
| **NB-03** | **Passive Extraction（受動的抽出）** | 情報を一方的に取り出すだけで矛盾検出をしない | ソース間の相違・矛盾を見逃す | 「ソース間で矛盾する主張を特定して」という指示を追加 |
| **NB-04** | **Monolingual Assumption（単一言語前提）** | 多言語ソースで言語切り替えが発生 | 出力が意図しない言語に切り替わる | 「全出力を日本語で」等、出力言語を明示 |
| **NB-05** | **Feature Misuse（機能の誤用）** | Audio OverviewでQ&A形式を期待、チャットでポッドキャスト形式を期待 | 期待と出力の乖離、不満足な結果 | 各機能の用途を理解: Audio=対話形式の解説、Chat=質問応答 |

## 4. 反復改善の罠

```
REFINE ループの失敗:

  ❌ Multi-Variable Change（複数変数同時変更）:
    → 1回の調整でオーディエンス・トーン・構成を同時に変更
    → どの変更が品質改善に寄与したか判別不能
    → 対策: 1回の調整で1つの変数だけ変更

  ❌ Endless Iteration（無限反復）:
    → 完璧を追求し5回以上の再生成を繰り返す
    → 時間の浪費、品質の限界に到達していることを認識しない
    → 対策: 最大3ラウンドのルール、スコア4.0以上で完了

  ❌ Source Blame Avoidance（ソース品質回避）:
    → 3回反復しても改善しないのにプロンプトだけ調整し続ける
    → ソース品質が根本原因なのに対処しない
    → 対策: 3回改善なし → ソース品質を見直す

  ❌ No Baseline Comparison（ベースライン比較なし）:
    → 改善前の出力を保存せずに調整
    → 改善したかどうか客観的に判断不能
    → 対策: A/B比較: Version A を保存してから Version B を生成

  ❌ Gut-Feel Evaluation（感覚的評価）:
    → 「何となく良くなった」で判断
    → 主観的で再現性のない評価
    → 対策: 5軸ルーブリック（Accuracy/Audience Fit/Engagement/Completeness/Actionability）を使用
```

## 5. プロンプトパターン誤用

```
パターン適用の罠:

  ❌ Pattern Stacking（パターン積み重ね）:
    → 全6パターンを同時適用
    → プロンプトが肥大化し指示が競合
    → 対策: 目的に応じて2-3パターンを選択的に使用

  ❌ Template Lock-In（テンプレート固定）:
    → 全コンテンツに同一テンプレートを機械的に適用
    → フォーマット・オーディエンスに合わないプロンプト
    → 対策: テンプレートはベースライン、コンテキストに合わせてカスタマイズ

  ❌ Over-Steering（過度なステアリング）:
    → AIの創造性を完全に制約する詳細すぎる指示
    → 機械的で生命力のない出力、予想外の洞察が失われる
    → 対策: 方向性は示すが空間を残す「ステアリング＝ガイド」の原則
```

## 6. Prism との連携

```
Prism での活用:
  1. STEER フェーズで SP-01〜07 のスクリーニング
  2. PREPARE フェーズでプロンプト構造の品質チェック
  3. REFINE フェーズで反復改善の罠を回避
  4. EVALUATE フェーズでルーブリックベースの客観評価を徹底

品質ゲート:
  - 「良い感じに」等の曖昧指示 → 具体的制約への置換（SP-01 防止）
  - プロンプト150語超 → 不要情報の除去提案（SP-02 防止）
  - 1プロンプトに複数タスク → 分割提案（SP-03 防止）
  - オーディエンス未指定 → Audience Anchor追加（SP-04 防止）
  - 除外指示なし → Negative Space追加（SP-05 防止）
  - ステアリングをチャット欄に入力 → Customizeボックスへの誘導（SP-07 防止）
  - 3回反復で改善なし → ソース品質レビュー提案（Source Blame Avoidance 防止）
```
