## 1. CI 成熟度モデル

### 4段階モデル（SCIP ベース）

| レベル | 名称 | 特徴 | Compete での対応 |
|--------|------|------|----------------|
| L1 | **Ad-hoc / Reactive** | リクエストベースの散発的 CI。体系的プロセスなし | MAP フェーズの基本実行 |
| L2 | **Defined / Structured** | 専任チーム、定期レポーティング、基本ツール | 全フレームワーク（MAP→ANALYZE→DIFFERENTIATE）の定期実行 |
| L3 | **Proactive / Integrated** | CI が戦略的意思決定に組み込まれ、複数部門で活用 | SHARPEN + Helm/Spark 連携 |
| L4 | **Advanced / Predictive** | AI/ML 予測分析、リアルタイム監視、自動化配信 | Multi-Engine Mode + 自動アラート |

### 成熟度評価軸

- **プロセス**: CI サイクルの標準化度合い
- **テクノロジー**: AI/自動化ツールの導入レベル
- **組織**: 専任チームの有無、クロスファンクショナル連携
- **文化**: CI がどの程度意思決定に活用されているか
- **測定**: ROI、予測精度、実行可能性率の追跡

## 2. AI × CI の融合（2024-2026）

### 主要統計

| 指標 | データ |
|------|--------|
| CI チームの AI 導入増加率 | **前年比 76%** |
| AI を日常的に使用する CI チーム | **60%** |
| CI への投資増加 | **前年比 24%** |
| CI により意思決定改善を実感 | **91%** |
| 競合案件でのウィンレート向上報告 | **77%** |

### AI が変える CI の4領域

| 領域 | Before | After |
|------|--------|-------|
| **データ収集** | 手動 Web リサーチ（8-12時間/月） | AI セマンティック監視（85-95% 削減） |
| **バトルカード** | 静的 PDF（30日で陳腐化） | 動的リアルタイム更新 |
| **アラート** | キーワードベース | セマンティック分析 + 自動要約 |
| **配信** | レポート共有 | CRM 統合 + Slack/Teams 自動配信 |

### 動的バトルカードの進化

```
従来: 手動作成 → PDF 共有 → 30日で陳腐化 → 営業が使わない
  ↓
現代: 50+ ソースから自動生成 → CRM サイドバー表示 → 継続的精度 → 利用率追跡
```

**効果**:
- 競合調査時間 **65% 削減**
- バトルカード参照率と受注相関の自動追跡
- 最も参照されるセクション・回答の可視化

## 4. CI サイクルの最新モデル

### 6ステップサイクル

```
1. Planning & Direction（計画・方向付け）
   → インテリジェンスニーズの明確化。戦略目標との整合
2. Collection（収集）
   → 一次情報（インタビュー）+ 二次情報（プレス、決算、レポート）
3. Processing（処理）
   → 非構造化データの構造化。AI フィルタリング・分類
4. Analysis（分析）
   → SWOT、Porter's Five Forces、ポジショニングマップ
5. Dissemination（配信）
   → 適切なステークホルダーへのタイムリーな知見配信
6. Feedback（フィードバック）
   → プログラムの有効性評価と次サイクルへの改善
```

### Compete フレームワークとのマッピング

| CI サイクル | Compete フェーズ |
|------------|----------------|
| Planning | 分析タイプ・競合選定 |
| Collection | MAP（intelligence-gathering.md） |
| Processing | MAP → ANALYZE 間のデータ構造化 |
| Analysis | ANALYZE（analysis-templates.md） |
| Dissemination | DIFFERENTIATE + ハンドオフ |
| Feedback | SHARPEN（intelligence-calibration.md） |

## 5. リアルタイム競合モニタリング

### 監視対象と頻度

| 対象 | 頻度 | ツール例 |
|------|------|---------|
| Web サイト変更 | リアルタイム | Crayon, Visualping |

### 自動化ワークフロー

```
競合 Web 変更検知 → AI 要約生成 → 重要度判定

  → Medium: 週次ダイジェスト
  → Low: 月次レポート
```
