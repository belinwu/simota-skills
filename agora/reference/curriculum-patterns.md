# Curriculum & Lesson Patterns

Read when building curriculum maps, sequencing modules, or authoring lesson plans and materials.

## Curriculum Map

A curriculum map shows how modules build toward outcomes. Include per module:
- Module ID + title
- Objectives covered (`LO-IDs`)
- Prerequisites (earlier module IDs or external)
- Estimated effort (hours)
- MVP vs stretch marking
- Assessment that closes the module

## Sequencing Patterns

| Pattern | Use when |
|---------|----------|
| **Linear / spiral** | Concepts build cumulatively; revisit core ideas at increasing depth (spiral) |
| **Scaffolded** | Heavy support early (worked examples), fading to independent practice |
| **Problem-first (PBL)** | Motivated learners; anchor each module in an authentic problem |
| **Just-in-time** | Practitioners; teach a concept right before it is needed in a project |

Respect prerequisite chains. Mark the **MVP path** (minimum modules to reach the core outcome) separately from **stretch/optional** modules so learners are never blocked by aspirational content.

## Weekly Plan

Break the time budget into weeks. Per week include: focus topic, concrete learning tasks, exercises, review/spaced-repetition slot, and the checkpoint (if any). Keep weekly load within the stated hours/week.

## Lesson Plan Structure

Each lesson plan includes:
1. **Objectives** — the `LO-IDs` this lesson serves.
2. **Timing breakdown** — minutes per segment (hook / instruction / practice / wrap).
3. **Activities** — what the learner does (not just what is presented). Favor active over passive.
4. **Materials** — readings, slides, datasets, tools required.
5. **Check for understanding (CFU)** — a quick formative probe before moving on.
6. **Common stumbling point** — the likely failure mode + remedy for this lesson.

## Material Design

| Artifact | Contents |
|----------|----------|
| `reading_list.md` | Curated sources; mark required vs optional; one-line "why read this" each |
| `exercises.md` | Graded difficulty; each maps to an `LO-ID`; include expected outcome |
| `assignments.md` | Larger applied tasks with deliverable + acceptance criteria |
| `project_briefs.md` | Capstone-style briefs: goal, constraints, deliverable, evaluation pointer |
| `glossary.md` | Domain terms with plain-language definitions; first-use anchors |

Every material maps to at least one objective. Exercises should span Bloom's levels appropriate to the stage, not only recall. Pre-empt the "happy path only" trap: include exercises that surface boundary cases and common errors.

## Engagement & Retention

- Interleave practice across topics rather than blocking, to improve transfer.
- Schedule spaced review (review_schedule.md) — expanding intervals for durable recall.
- Tie each module to an authentic application so motivation stays concrete.
