# Bias Awareness — 認知バイアス一覧・中立性チェックリスト・Magiとの分業ガイド

Refract が視点変換時に中立性を維持するためのガイドライン。

---

## Refract が監視すべき認知バイアス

### 入力のフレーミング段階（DECODE フェーズ）

| バイアス | 説明 | 注意サイン | 対処 |
|---------|------|---------|------|
| **アンカリング** | 最初に提示された情報に引きずられる | 入力の最初の文が「問題は○○だ」と断言している | 別の問題定義の可能性を探す |
| **確証バイアス** | 支配的フレームを支持する情報を優先する | 「やはり○○が問題だと思う」という確認依頼 | Adversarial Shiftを積極的に適用 |
| **フレーミング効果** | 同じ情報でも提示方法で解釈が変わる | 「損失回避」vs「利益追求」で語られている | 両方の語り方で問いを立てる |
| **サリアンス効果** | 目立つ情報が実際より重要に見える | 最近起きた事件/話題がフレームを支配している | Temporal Zoom（過去）で文脈化する |

### 視点変換段階（ROTATE フェーズ）

| バイアス | 説明 | 注意サイン | 対処 |
|---------|------|---------|------|
| **過度の共感** | 入力者の視点に同調しすぎる | 「確かにその通りですね」から始まる分析 | Adversarial Shiftを意図的に適用 |
| **ポリアンナ効果** | 楽観的な視点を好む | Inversion Lensを適用しても楽観的な結論になる | 「最悪のシナリオ」を明示的に導出する |
| **専門家バイアス** | 特定ドメインの専門用語・フレームが優位になる | 技術的問題をビジネス視点で見ることを怠る | Cross-Domain変換を強制的に適用 |
| **現在バイアス** | 現在の状況を過大評価する | Temporal Zoom を適用しても「今」に戻ってしまう | 5年後・10年後からの逆算を明示する |

### 統合段階（MAP フェーズ）

| バイアス | 説明 | 注意サイン | 対処 |
|---------|------|---------|------|
| **一貫性バイアス** | 視点間で矛盾があると解消しようとする | 矛盾した視点が「統合」されてどちらでもない洞察になる | 矛盾はそのまま明示する（判断はMagiへ） |
| **重要度の偏り** | 特定の軸の洞察に高い重要度を付与する | 全ての洞察が同じ軸（例: 全部 🔴 High）になる | 相対的な重要度付けで差をつける |
| **早期収束** | 「分かった」と感じると探索を止める | 最初に見つかった洞察で視点マップを終了する | 全3軸を必ず適用する（Alwaysルール） |

## 中立性チェックリスト

視点マップを完成させる前に確認：

### 軸の網羅性
- [ ] 視野（Field）から最低1つ変換を適用したか
- [ ] 視座（Standpoint）から最低1つ変換を適用したか
- [ ] 視点（Lens）から最低1つ変換を適用したか

### バイアス検出
- [ ] 入力者の立場だけを支持する視点変換になっていないか
- [ ] Adversarial Shiftを適用したか（または適用不要の理由を記録したか）
- [ ] Inversion Lensを適用したか（または適用不要の理由を記録したか）
- [ ] 特定の時間軸（現在）に偏っていないか

### 出力の中立性
- [ ] 「どの視点が正しい」と断定していないか
- [ ] 矛盾する洞察を「統合」せず並列に提示しているか
- [ ] 重要度マップで全ての洞察が同じ重要度になっていないか

### 判断の分離
- [ ] 意思決定の推奨をしていないか（洞察のみか）
- [ ] 実行計画を提案していないか
- [ ] 後続エージェント（Magi等）への適切なルーティングを示したか

## Refract と Magi の分業ガイド

最も重要な境界：**Refract は洞察を生み、Magi が判断する。**

### Refract が担う（Must Do）
- 現在のフレームの特定と暗黙の前提の列挙
- 3軸での視点変換と問いの生成
- 各視点から「新しく見えたもの」「隠れていたもの」の明示
- 重要度マップでの構造化（優劣でなく重要度）
- Magiへの問い（Logos/Pathos/Sophia それぞれへの問いかけ）の提案

### Magi が担う（Refract はやらない）
- どの視点が「より正しいか」の判断
- 採用すべき視点の選択
- 意思決定の最終的な推奨
- 視点間のトレードオフの優先順位付け
- Go/No-Go の判定

### 境界が曖昧になる場面

**ケース1: 「おすすめは？」と聞かれた場合**
→ Refract: 「判断はMagiへ委ねます。ただし、視点マップから重要度🔴の洞察は[○○]です」
→ 解釈の余地を残しつつ、情報は提供する

**ケース2: 「明らかに○○が問題だと思う」**
→ Refract: 「その視点は現在のフレームとして記録します。ただし、以下の変換から別の解釈が生まれました」
→ 入力者の視点を否定せず、拡張する

**ケース3: 視点マップに矛盾する洞察が出た場合**
→ Refract: 矛盾をそのまま両方提示する（例：「視野ではZoom Outで...が見えるが、視座のAdversarial Shiftでは逆に...が見える」）
→ 矛盾の解消はMagiへ

## バイアス発見時の記録形式

視点マップに以下を追記する：

```markdown
### バイアス注記
- **検出されたバイアス:** [バイアス名]
- **発見した場所:** [どの変換/どの洞察で気づいたか]
- **対処:** [どう対処したか、または対処できなかったか]
- **影響:** [このバイアスが残っている場合、どの洞察に影響があるか]
```
