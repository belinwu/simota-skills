# Self-Analysis Kit

Prompts and structures for strengths, values, work-style, and the achievement inventory. Grounding rule: everything must trace to real, verifiable experience — never fabricate a track record.

## Strengths & Weaknesses

- List candidate strengths, then validate each with a concrete past situation where it produced a result.
- Distinguish *learned skills* (improvable) from *natural inclinations* (leverage-able).
- For weaknesses: separate "fixable gap for target role" (→ learning plan) from "delegate/avoid" (→ route/role fit).

## Values

Rank what the user optimizes for: autonomy, impact, compensation, stability, learning, flexibility, recognition, mission. Surface conflicts (e.g., max compensation vs max autonomy) — these constrain viable routes.

## Work-Style

Capture: remote/hybrid/onsite preference, solo vs team, deep-work vs collaboration ratio, management vs IC track, hours/intensity tolerance, avoid-list (work patterns to rule out).

## Achievement Inventory

Use **STAR-Impact** rows:

| # | Situation | Task | Action | Result (quantified) | Skill demonstrated |
|---|-----------|------|--------|---------------------|--------------------|

Rules:
- Quantify results where data exists; mark estimates as estimates.
- Each row maps to a skill — this inventory feeds positioning evidence, resume bullets, and interview stories.
- Reject vanity entries (titles, tenure) with no demonstrated outcome.

## Output

A `strengths_weaknesses` doc, `values` doc, `work_style` doc, and `achievement_inventory` table. These ground all downstream positioning and assets — produce them before market/positioning work.
