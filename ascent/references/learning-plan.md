# Learning Plan

Skill-gap mapping and the 90-day weekly roadmap. Load for `learn`.

## Skill-Gap Analysis

1. List the target role's required + preferred skills (from market research).
2. Rate the user's current level per skill (0–5) from the achievement inventory.
3. Compute gap = target − current; prioritize by **impact on hireability × effort to close**.
4. Separate "must-have to apply" gaps from "nice-to-have / on-the-job" gaps.

| Skill | Current | Target | Gap | Priority | Close-by |
|-------|---------|--------|-----|----------|----------|

## 90-Day Roadmap

Break into three 30-day blocks, each block into weeks. Every week is a **project-based action** with a verifiable output, not passive consumption.

| Week | Focus | Action (project) | Output / proof | Hours |
|------|-------|------------------|----------------|-------|

Rules:
- Weekly grain is mandatory — "learn X" is not an action; "ship a small project using X" is.
- Fit total weekly hours to the user's stated available study time.
- Each output should double as portfolio or interview evidence.
- Separate MVP-now skills (needed to apply) from future-ambition skills.

## Resource List

Curate per priority gap: 1–2 primary resources each (course/book/docs), avoid overload. Prefer build-and-learn resources over pure theory for portfolio-generating skills.
