| 開発フェーズ | 季節 | 季語的表現 (JP) | Expression (EN) |

## Negative Event Transformation（ネガティブイベントのポジティブ変換）

ネガティブなイベントを詩的にポジティブに変換するパターン。

| ネガティブイベント | 変換パターン | 変換例 |
|-----------------|------------|--------|
| バグ発見 | 発見→治癒の物語 | 「隠れし傷を見つけ出し、手当てを施す職人の手」 |
| リバート | 戦略的撤退 | 「退くも勇気、進むも勇気」 |
| 障害発生 | 試練→克服の英雄譚 | 「嵐の夜を越え、朝日を迎える」 |
| テスト失敗 | 学びの機会 | 「転びて初めて知る道の形」 |
| 大規模コンフリクト | 和解と統合 | 「激しき議論の果てに生まれし調和」 |
| 技術的負債 | 冬を越す準備 | 「雪の下に眠る種は、春を待つ」 |
| デッドライン超過 | マラソンの最終コーナー | 「長き道のりも、あと一歩」 |
| 機能削除 | 選択と集中 | 「枝を落として幹を育てる」 |

### Transformation Principles

1. **事実は変えない**: 障害を「良かったこと」にはしない。だが「乗り越えた」と詠む
2. **人を責めない**: 「誰かのミス」ではなく「チームの試練」として描く
3. **学びを見出す**: すべてのネガティブイベントには成長の種がある
4. **未来への希望**: 過去の困難から未来への期待を紡ぐ
5. **ユーモアは慎重に**: 軽い自嘲は可、だが他者への皮肉は不可
