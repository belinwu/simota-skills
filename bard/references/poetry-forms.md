# Poetry Forms Reference

Bard が使用する詩形の詳細ルール、構造、使用条件。

---

## Japanese Forms（日本語詩形）

### 俳句 (Haiku)

**構造:** 5-7-5（17音）
**用途:** 単一イベント、短いスプリントのハイライト、瞬間の印象

**ルール:**
- 3行構成: 5音 / 7音 / 5音
- 季語的表現を含める（開発の季節感）
- 切れ字（や、かな、けり）で余韻を残す
- 一つのコミット/PR/イベントに焦点を当てる

**音数カウント:**
- 長音（ー）は1音
- 促音（っ）は1音
- 撥音（ん）は1音
- 拗音（きゃ等）は1音

**テンプレート:**
```
[5音: 状況・季節の描写]
[7音: 動作・変化の核心]
[5音: 結果・余韻]
```

**使用条件:**
- コミット数: 1-5
- イベント: 単一の特徴的な変更
- 言語: 日本語推奨（英語も可能だが5-7-5 syllablesに変換）

---

### 短歌 (Tanka)

**構造:** 5-7-5-7-7（31音）
**用途:** 感情を込めた振り返り、個人の貢献を称える

**ルール:**
- 5行構成: 5音 / 7音 / 5音 / 7音 / 7音
- 上の句（5-7-5）: 客観的な描写
- 下の句（7-7）: 主観的な感情・感想
- 俳句より深い感情表現が可能

**テンプレート:**
```
[5音: 情景・状況]
[7音: 動作・変化]
[5音: 転換点]
[7音: 感情・影響]
[7音: 余韻・未来への展望]
```

**使用条件:**
- コミット数: 1-10
- イベント: 感情的な重みがある変更（大きなバグ修正、初リリースなど）
- 言語: 日本語

---

### 連歌 (Renga) / 俳句コレクション

**構造:** 複数の俳句を連ねる形式
**用途:** スプリント振り返り、複数イベントの俯瞰

**ルール:**
- 各俳句は独立して成立すること
- 前の句から次の句へのテーマの流れ（付け合い）を意識する
- 通常3-7句で構成
- 最初の句（発句）が最も重要

**テンプレート:**
```
[発句: スプリントの象徴的な出来事]
[脇句: 関連するもう一つの変化]
[第三: 視点の転換]
[...追加の句...]
[挙句: 締めくくり・展望]
```

**使用条件:**
- コミット数: 3-15
- イベント: スプリント全体、複数の関連変更
- 言語: 日本語

## English Forms（英語詩形）

### Free Verse（自由詩）

**構造:** 固定なし
**用途:** 複雑なストーリー、オンボーディング、長期の振り返り

**ルール:**
- 韻律・行数の制約なし
- 自然なリズムと改行で構成
- ストーリーアーク（導入→展開→クライマックス→結末）を持つ
- 段落/スタンザで論理的に区切る

**構成ガイドライン:**
- 導入（1-3行）: 時間・場所の設定
- 展開（5-15行）: 主要な出来事
- クライマックス（2-5行）: 最も重要な瞬間
- 結末（2-4行）: 振り返り・展望

**使用条件:**
- コミット数: 10+
- イベント: 複雑な物語、プロジェクト全体
- 言語: 英語推奨（日本語も可能）

---

### Ballad（バラッド）

**構造:** 4行スタンザ（ABAB or ABCB 韻）
**用途:** バグ退治の物語、障害対応の記録

**ルール:**
- 4行1スタンザ
- 2行目と4行目で韻を踏む（ABCB）、または交互韻（ABAB）
- ストーリーテリング形式（物語を語る）
- 3-6スタンザが標準

**テンプレート:**
```
[A: 状況の設定]
[B: 問題の発見]
[C: 行動の描写]
[B: 結果（Bと韻を踏む）]
```

**使用条件:**
- コミット数: 5-20（物語性のある変更群）
- イベント: バグ修正、障害対応、冒険的な変更
- 言語: 英語推奨

---

### Epic（叙事詩）

**構造:** 長編自由詩 / セクション分割
**用途:** リリース祝詩、プロジェクト起源の物語

**ルール:**
- 複数のセクション（章）で構成
- 各セクションにサブタイトル
- 英雄（開発者/チーム）の旅路を描く
- 冒頭に「歌い出し」（invocation）を含める

**構成:**
1. **Invocation（歌い出し）**: テーマの宣言
2. **Genesis（起源）**: プロジェクト/機能の始まり
3. **Trials（試練）**: 困難・バグ・課題
4. **Triumph（勝利）**: 解決・完成
5. **Legacy（遺産）**: 成果の意義・未来

**使用条件:**
- コミット数: 20+
- イベント: メジャーリリース、プロジェクト全体の回顧
- 言語: 英語推奨（日本語も可能）

---

### Sonnet-style（ソネット風）

**構造:** 14行（変形可）
**用途:** リリース祝詩、凝縮された物語

**ルール:**
- 14行構成（厳密な韻律は不要）
- 3つのカトレーン（4行）+ 1つのカプレット（2行）
- 最後のカプレットで結論・転換
- シェイクスピア型を参考にするが、厳密なiambic pentameterは不要

**テンプレート:**
```
[カトレーン1: 状況・問題の提示]（4行）
[カトレーン2: 展開・試行]（4行）
[カトレーン3: クライマックス・解決]（4行）
[カプレット: 結論・メッセージ]（2行）
```

**使用条件:**
- コミット数: 5-30
- イベント: リリース、マイルストーン達成
- 言語: 英語

## Form Selection Matrix

コミット数と目的に基づく詩形選択ガイド。

| コミット数 | 目的 | 推奨詩形 | 代替 |
|-----------|------|---------|------|
| 1 | 単一イベント | 俳句 | 短歌 |
| 2-5 | 小規模振り返り | 俳句コレクション | 短歌 |
| 3-10 | スプリント振り返り | 連歌 | 自由詩 |
| 5-15 | バグ退治の物語 | バラッド | 自由詩 |
| 5-30 | リリース祝詩 | ソネット風 | 叙事詩 |
| 10+ | 複雑な物語 | 自由詩 | 叙事詩 |
| 20+ | プロジェクト全体 | 叙事詩 | 自由詩 |
| 任意 | 感情的深さ重視 | 短歌 | 自由詩 |
| 任意 | 国際チーム向け | Free Verse | Ballad |

### Language Selection

| 詩形 | 推奨言語 | 理由 |
|------|---------|------|
| 俳句 | 日本語 | 5-7-5の音韻は日本語で最も美しい |
| 短歌 | 日本語 | 和歌の伝統に根差す |
| 連歌 | 日本語 | 俳句の連なりとして日本語が自然 |
| 自由詩 | どちらでも | 制約がないため言語を選ばない |
| バラッド | 英語 | 英語の韻律が活きる |
| 叙事詩 | 英語 | 長編の物語は英語が読みやすい場合が多い |
| ソネット風 | 英語 | シェイクスピアの伝統に根差す |

## Quality Checklist

生成した詩の品質チェック:

- [ ] 選択した詩形のルール（音数、行数、構造）を守っているか
- [ ] 実際のgitデータに基づいているか（架空の情報がないか）
- [ ] 技術用語が詩的に変換されているか（生のコード/ファイル名の羅列でないか）
- [ ] 感情的な深みがあるか（単なるリスト化でないか）
- [ ] 対象者（チーム、個人、新人）に適した表現か
- [ ] ネガティブな表現が個人を攻撃していないか
- [ ] タイトル、ソースサマリー、詩形ラベルが含まれているか
