# Research Ops & Democratization

> ResearchOps 成熟度、Atomic Research、インサイト管理、リサーチ民主化フレームワーク

## 1. ResearchOps の現状（2025-2026）

### 主要統計

| 指標 | データ |
|------|--------|
| PwDR（People who Do Research）がいる組織 | **71%** |
| リーダーシップがリサーチ価値を理解 | **58%**（同僚は 74%） |
| 専任 ResearchOps サポートなしの企業リサーチャー | **51%** |
| ソロリサーチャーが 1,000人未満の企業に所属 | **78%** |
| ResearchOps のスコープが拡大 | **17/21** 参加者（リソースは横ばいまたは減少） |
| リサーチの将来に楽観的 | わずか **11%** |

### 2025年の5大テーマ（Rally UXR 調査）

| テーマ | 内容 |
|--------|------|
| **AI 支配** | AI がシステムに統合。「プログラムとして意図的な構造化」が必要 |
| **組織再編** | 成熟チームの統合 vs 新規チームの積極的採用。UXR と市場調査の統合傾向 |
| **民主化進化** | ResearchOps は維持、リサーチャー職は削減。非リサーチャーの調査支援が課題 |
| **雇用危機** | レイオフ継続。「AI 経験」が求人要件に（未成熟なスキルセット） |
| **コミュニティ** | 孤立リサーチャーの生命線。「1人チームが200人チームに」 |

## 2. ResearchOps 成熟度モデル

### 4段階モデル

| レベル | 名称 | 特徴 | Researcher での対応 |
|--------|------|------|-------------------|
| L1 | **Ad-hoc** | リサーチは散発的。標準プロセスなし | DEFINE 基本実行 |
| L2 | **Emerging** | 専任リサーチャー配置、基本ツール導入 | 全フレームワーク実行 |
| L3 | **Scaling** | ResearchOps 専任、リポジトリ構築、民主化プログラム | DISTILL + エコシステム連携 |
| L4 | **Strategic** | リサーチが戦略的意思決定に組み込まれ、AI 自動化活用 | 全フェーズ + AI ハイブリッド |

### 成熟度評価チェックリスト

**インフラ:**
- [ ] リサーチリポジトリが存在するか?
- [ ] 参加者パネル/リクルーティングシステムがあるか?
- [ ] ツールセットが標準化されているか?
- [ ] 同意書・プライバシーポリシーがテンプレート化されているか?

**プロセス:**
- [ ] リサーチのリクエスト→実施→共有のフローが定義されているか?
- [ ] インサイトの分類・タグ付けルールがあるか?
- [ ] 品質チェック（バイアスレビュー等）が標準化されているか?

**組織:**
- [ ] リサーチの価値をリーダーシップが理解しているか?
- [ ] クロスファンクショナルな連携があるか?
- [ ] 民主化プログラムのガバナンスがあるか?

## 3. Atomic Research

### 概念

リサーチのサイロ化を解消し、部族的知識の影響を緩和する構造化されたインサイト管理手法。

### 4層構造

```
Experiments（実験）
  → Facts（事実）: 観察可能なデータポイント
    → Insights（インサイト）: 複数の事実からの推論
      → Recommendations（推奨）: アクション可能な提案
```

### Atomic Research の原則

| 原則 | 説明 |
|------|------|
| **粒度** | インサイトを最小単位（atomic）に分解 |
| **再利用性** | 1つのインサイトが複数のレポート・判断に使用可能 |
| **トレーサビリティ** | 推奨 → インサイト → 事実 → 実験まで遡れる |
| **検索性** | タクソノミーに基づくタグ付けで発見可能 |

### リサーチリポジトリ設計

```markdown
## Research Repository Structure

### タクソノミー例
- **テーマ**: オンボーディング / 検索 / 決済 / サポート
- **セグメント**: 新規ユーザー / パワーユーザー / 管理者
- **研究種別**: 探索的 / 評価的 / 生成的
- **信頼度**: High / Medium / Low
- **鮮度**: 3ヶ月以内 / 6ヶ月以内 / 1年以上（要更新）

### インサイトカード（Atomic 版）
| フィールド | 内容 |
|-----------|------|
| Insight ID | INS-2026-001 |
| ステートメント | [1文でのインサイト] |
| 根拠 Facts | FACT-001, FACT-003, FACT-007 |
| 元実験 | EXP-2026-Q1-003 |
| セグメント | [対象ユーザー] |
| 信頼度 | High / Medium / Low |
| 鮮度 | [最終検証日] |
| 関連推奨 | REC-2026-001 |
```

## 4. リサーチ民主化

### 民主化の進化（2025-2026）

```
旧: 「全員がリサーチャーになるべき」
  ↓
新: 「Customer Discovery Enablement」（顧客発見支援）
    + Strategic Research（戦略的リサーチ）の分離
```

### 民主化ガバナンスフレームワーク

| レベル | 誰が | 何を | ガバナンス |
|--------|------|------|----------|
| **Self-service** | PM / デザイナー / エンジニア | アンモデレート・ユーザビリティテスト、簡易サーベイ | テンプレート + ガイドライン |
| **Guided** | PwDR + リサーチャーの助言 | モデレートインタビュー、テーマ分析 | リサーチャーがレビュー |
| **Expert-led** | 専任リサーチャー | 戦略的リサーチ、複雑な混合手法 | 完全リサーチプロセス |

### 民主化リスクと対策

| リスク | 対策 |
|--------|------|
| 方法論の誤り（非リサーチャーによる実施） | テンプレート + チェックリスト + レビュープロセス |
| AI を使った未検証の結果 | AI 出力の人間レビューを必須化 |
| 「リサーチシアター」化 | リサーチの目的と使用計画を事前に明確化 |
| インサイトの断片化 | 統一リポジトリへの登録を必須化 |
| バイアスの増大 | bias-checklist.md の簡易版を PwDR に提供 |

### PwDR 向けセルフサービスキット

```markdown
## Self-Service Research Kit

### 実施可能なリサーチ
- [ ] アンモデレート・ユーザビリティテスト（5-6名）
- [ ] 短いサーベイ（10問以内）
- [ ] A/B テストの定性フォローアップ
- [ ] 競合プロダクトのヒューリスティック評価

### 実施前チェック
- [ ] リサーチ目的を1文で書けるか?
- [ ] 結果をどう使うか明確か?
- [ ] 同意書テンプレートを使用したか?
- [ ] リサーチャーにレビュー依頼したか?

### 実施不可（リサーチャーに依頼）
- 探索的インタビュー
- センシティブなトピック
- 脆弱な集団の調査
- 大規模な混合手法研究
```

## 5. インサイト管理プラットフォーム比較

### 主要ツール

| ツール | 特徴 | 対象 |
|--------|------|------|
| **Dovetail** | リサーチリポジトリ + AI 分析 + タグ付け | 中〜大企業 |
| **Rally** | パネル管理 + ResearchOps 自動化 | ResearchOps チーム |
| **Condens** | Atomic Research 対応リポジトリ | リサーチ成熟企業 |
| **EnjoyHQ** | インサイトハブ + ステークホルダー共有 | 民主化推進企業 |
| **Notion/Airtable** | カスタマイズ可能な軽量リポジトリ | スタートアップ |

### 選択指針

| 条件 | 推奨 |
|------|------|
| リサーチチーム5名以上 + 民主化推進 | Dovetail / Condens |
| ソロリサーチャー + 軽量運用 | Notion + テンプレート |
| ResearchOps 専任あり + パネル管理必要 | Rally |
| 既存ツールへの統合重視 | EnjoyHQ |
