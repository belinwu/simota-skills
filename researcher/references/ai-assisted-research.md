# AI-Assisted Research & Synthetic Users

> AI 活用リサーチ手法、Synthetic Users のリスクと限界、AI×人間のハイブリッドアプローチ

## 1. AI × UX リサーチの現状（2025-2026）

### 主要統計

| 指標 | データ |
|------|--------|
| AI 活用分析・合成をトップトレンドと回答 | **88%**（100人の UX リサーチャー調査） |
| Synthetic Users をインパクトありと回答 | **48%** |
| AI による定性分析時間の削減 | **最大 80%** |
| AI を日常的にリサーチで使用 | 全リサーチャーの大多数 |
| AI 活用に懐疑的なリサーチャー | 一部は「遅れをとる」と予測 |

### AI の現在の利用領域

| 用途 | 利用率 |
|------|--------|
| 分析・パターン発見 | **23%** |
| 文字起こし・ノートテイキング | **21%** |
| デリバラブル作成 | **13%** |
| インサイト合成 | **8%** |
| リサーチ資料作成 | **7%** |
| AI 未使用 | **5%** |
| その他 | **23%** |

## 2. AI が変える4つのリサーチ領域

| 領域 | Before | After（AI 活用） |
|------|--------|-----------------|
| **文字起こし** | 手動テープ起こし（数時間/セッション） | 自動文字起こし + 話者識別 |
| **コーディング** | 手動 in-vivo コーディング | セマンティック自動コーディング + テーマ提案 |
| **パターン検出** | 複数セッション横断の手動比較 | AI パターン認識 + 異常値検出 |
| **レポーティング** | 手動レポート作成 | ドラフト自動生成 + ハイライト抽出 |

### AI の役割分担原則

```
AI が担う「What」: データ処理、パターン認識、構造化
  ↕
人間が担う「Why」と「How」: 共感、戦略、判断
```

**鍵**: AI は「テディアスな作業」を自動化し、リサーチャーはより創造的・戦略的な仕事に集中。

## 3. Synthetic Users（合成ユーザー）

### 定義

LLM ベースで生成されたユーザープロファイル。実際の人間を調査せずに人工的なリサーチ結果を提供。大規模データセットで訓練されたモデルに基づき、思考・ニーズ・体験をシミュレート。

### 精度データ

- Stanford 研究: 合成エージェントは人間の回答を **最大 85% の精度** で模倣可能
- ただし ChatGPT はツリーテストで実ユーザーを「大幅に上回る」成績 → **実際の行動を再現できない**

### Synthetic Users の DO / DON'T（NN/g 推奨）

| DO（推奨） | DON'T（非推奨） |
|-----------|---------------|
| 実ユーザー調査の**準備**として使用 | 実ユーザーリサーチの**代替**として使用 |
| 出力を**検証すべき仮説**として扱う | 合成結果を**本物のリサーチ**として提示 |
| 広く文書化されたユーザーグループに適用 | ニッチ・専門的な集団に使用 |
| プロトジャーニーマップの初期作成 | ステークホルダーがリサーチ代替と見なすリスクがある場合 |
| インタビューガイドのパイロットテスト | コンセプト検証のフィードバックとして依存 |

### Synthetic Users の根本的限界

| 限界 | 詳細 |
|------|------|
| **Sycophancy（追従性）** | AI は「喜ばせようとする」傾向 → すべてのアイデアが「良い」と評価される |
| **感情の不在** | フラストレーション・喜びを「感じる」ことができない |
| **行動データの欠如** | 実際のプロダクト使用や固有の生活体験をモデル化できない |
| **深さの欠如** | トレードオフ・文脈・実装の具体性なしに汎用リストを生成 |
| **組織リスク** | 合成リサーチへの依存が始まると、本物のユーザー中心設計への長期コミットメントが損なわれる |

### Synthetic Users 使用判断フローチャート

```
Q: 調査対象は広く文書化されたユーザーグループか?
  → No → Synthetic Users 不適切。実ユーザーリサーチ必須
  → Yes ↓
Q: 目的はデスクリサーチ・仮説生成・準備段階か?
  → No → Synthetic Users 不適切。実ユーザーリサーチ必須
  → Yes ↓
Q: ステークホルダーが「これはリサーチ代替」と誤解するリスクは?
  → High → 使用控え。誤解リスク > 効率メリット
  → Low → Synthetic Users を「仮説生成ツール」として限定使用
           → 必ず実ユーザーで検証する計画を立てる
```

## 4. AI 活用リサーチのリスク管理

### 主要リスク

| リスク | 影響 | 対策 |
|--------|------|------|
| **ハルシネーション** | 存在しないパターンの「発見」 | 必ず人間が元データで検証 |
| **バイアス増幅** | 訓練データのバイアスが結果に反映 | 多様なデータソースの使用 + バイアスチェック |
| **プライバシー** | 参加者データの AI 処理 | データ匿名化の徹底 + 処理範囲の限定 |
| **スキル低下** | AI 依存による分析スキルの退化 | 定期的な手動分析の実施 |
| **「人間味」の喪失** | 沈黙・抵抗・不快感の見落とし | 定性インタビューは人間が実施 |

### AI 活用チェックリスト

**AI 分析前:**
- [ ] 元データの品質を人間が確認したか?
- [ ] AI ツールへの入力に個人情報が含まれていないか?
- [ ] 分析目的と期待する出力を明確化したか?

**AI 分析中:**
- [ ] AI の出力を元データと照合しているか?
- [ ] 「意外な発見」を懐疑的に検証しているか?
- [ ] 複数の AI ツール/プロンプトで結果を比較しているか?

**AI 分析後:**
- [ ] 人間による最終レビューを実施したか?
- [ ] AI が見落としている文脈・ニュアンスを補完したか?
- [ ] AI 活用の範囲と限界をレポートに明記したか?

## 5. Researcher での活用指針

| Researcher フェーズ | AI 活用ポイント | 注意点 |
|-------------------|---------------|--------|
| DEFINE | 既存リサーチの AI 要約でスコーピング加速 | 要約の正確性を確認 |
| DESIGN | Synthetic Users でインタビューガイドをパイロット | 実ユーザーでの検証を必須化 |
| ANALYZE | AI コーディング + パターン検出で分析加速 | 元データとの照合必須 |
| SYNTHESIZE | AI ドラフト生成 → 人間が深化・文脈追加 | AI 出力を「最終版」にしない |
| DISTILL | AI で大量データの傾向分析 | キャリブレーションは人間判断 |
